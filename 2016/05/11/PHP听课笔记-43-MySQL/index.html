五、创建索引

1. 主键索引
    主要作用是确定数据库表里一条特定数据记录的位置

    最好为每一张数据表定义一个主键，是为了让此列没有重复值，加快查询速度。

    一个表只能指定一个主键

    主键的值不能为空


2. 唯一索引 --不是为了提高速度，而是为了避免重复。
    都可以防止创建重复的值

    每个表都可以有多个唯一索引

    unique



3. 常规索引 --快速的定位记录

    最重要的技术

    提升数据库的性能

    11111111111
    22222222222
    3333333333333
    44444444444444
    55555555555
    。。。
    101010100101011

    索引顺序  

  可以提高查找的速度， 减慢数据列上插入，删除， 修改

  可以单独使用，也可在创建表时创建，因为索引和表一样是独立的数据对象

  index 和key 是同义词，用哪个都可以。

  多列都可以，

4. 全文索引 --不使用匹配模式的情况下搜索关键词或索引。

    fulltext类型索引， MyISAM 表类型使用， 只有在varchar char text文本字符串上使用

    也可以多个数据列使用，会把所有关键词列成一份清单。

六、数据表类型及存储位置

MySQL和大多数数据库不同， MySQL有一个存储引擎概念。

MySQL可以针对不同的存储引擎需求可以选择最优的存储引擎

引擎 数据表类型

我们只学12中的MyISAM 和 InnoDB两个

MyISAM 表类型是默认的

__选择MyISAM还是选择InnoDB__

注意： 在一个MySQL库中可以（创建表时）指定不同表类型

MyISAM 表类型

成熟稳定、易于管理。是表格锁定的机制。强调快速读取操作

缺点：
1.要经常使用OPTIMIZE TABLE 表名，此命令用于恢复更新机制所浪费的空间。碎片整理。
2.有些功能不支持

InnoDB 表类型

    支持一些MyISAM所不支技功能

    缺点：占用空间大

功能        MyISAM               InnoDB

事务处理        不支持                 支持   
数据行锁定      不支持               支持
外键约束        不支持               支持
表空间占用    相对小               相对大 最大2倍
全文索引        支持                 不支持
