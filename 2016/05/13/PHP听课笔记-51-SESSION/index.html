PHP听课笔记-51-SESSION

发表于 2016-05-13   |   分类于 PHP

使用SESSION进行会话控制。

和cookie不同的是，session是保存在服务器端的。

session ID 保存在两个地方

基于COOKIE
基于URL

配置信息存在于服务器端的PHP配置文件。

基于COOKIE

和cookie不同，session必须先启动，把PHP核心程序有关session的核心变量预先放置于内存中。

—demo.php—

执行之后，session就注册到服务器端了。以后的每个页面都必须开启session

—two.php—

因为是基于cookie的，所以session ID 保存在客户端的cookie中了。

—destory—

如果不想每个页面都添加start，在PHP配置中可以更改session.auto_start = 1
启用后，不能将对象放到session中，因为类必须在session开启之前加载。

基于URL

不使用COOKIE依旧可用。
重点：将参数加到URL串上，并且通过SESSION_ID[__GET[“sid”]]方式，将sid确定为附属session的参数名。
PS：将sid更名为PHPSESSID后，此名就不用再进行命名了，因为是内置参数。

分类code/SID常量

能够根据浏览器属性，来自动切换基于cookie或基于url

重点：更改：session_id为常量SID；

code-login.php

再将各php文件中的相关项目中的echo session_id项更改为echo SID ，就能看到会根据浏览器自动选择方式了。
缺点;需要每个页面都要添加echo SID
改进：在PHP配置文件中，session.use_trans_sid = 1
这样就不需要添加SID了，只需要在login.php的前部分location写上即可。因为PHP不认识这种跳转。
会自动识别普通跳转、form以及header。
